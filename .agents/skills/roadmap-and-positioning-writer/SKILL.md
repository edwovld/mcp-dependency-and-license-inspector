# Roadmap & Positioning Writer

Исполнитель задач 6–7 из `plan/07-next-steps.md`: подготовка к защите проекта (Q&A для жюри) и стратегические материалы (roadmap, позиционирование, коммерциализация).

## Когда использовать

- Нужен список вопросов от жюри/аудитории с готовыми ответами (15–20 вопросов).
- Требуется roadmap v2 и v3 (по 3–5 фич на версию).
- Нужно competitive positioning — сравнение с Snyk, Dependabot, GitHub Advanced Security.
- Сценарии коммерциализации: использование в компаниях, метрики, отчёты для бизнеса.

## Plan 07 — задачи и артефакты (6–7)

| Задача | Артефакт | Краткое описание |
|--------|----------|------------------|
| 6 | `docs/jury-qa.md` | 15–20 вопросов жюри с ответами (2–5 предложений). Темы: архитектура, MCP/Python/детерминизм, ограничения, безопасность, сравнение с SCA, roadmap. |
| 7a | `plan/07-roadmap.md` или в `plan/07-next-steps.md` | Roadmap v2 и v3: новые экосистемы (Maven/Gradle), CI/CD, отчёты для compliance/юристов. |
| 7b | `docs/competitive-positioning.md` | Обзор SCA; таблица/список: где сильнее (MCP, детерминизм, DEMO без платных API), где слабее, что добавить. |
| 7c | (опционально) в том же файле | 2–3 сценария коммерциализации; метрики и отчёты для бизнеса. |

## Контекст для выполнения

- **Вход:** `plan/01–07`, `docs/product-summary.md`, `README`, `DEMO.md`, `docs/FAQ.md` (если есть), `plan/05-architecture-and-api.md` (tools и ресурсы).
- **Формат jury-qa:** номер, вопрос, ответ (2–5 предложений).
- **Формат roadmap:** нумерованные списки с краткими описаниями фич.
- **Формат positioning:** таблица или список; без воды.

## Инструкции

- **Jury Q&A:** типичные вопросы по архитектуре (почему MCP, Python, детерминизм), ограничениям (образ 500 MB, без LLM), безопасности, сравнению с Snyk/Dependabot/GitHub AS, планам развития.
- **Roadmap:** учитывать новые экосистемы, CI/CD, отчёты compliance/юристов.
- **Positioning:** честно отражать сильные (интеграция в MCP, детерминизм, DEMO без платных API) и слабые стороны.
- При неясных требованиях — уточнять у пользователя.

## Ограничения

- Не меняет код, контракты tools и логику MCP‑сервера.
- Результаты — только в `docs/` и `plan/`.
- Соблюдать стиль и терминологию из plan/01–07.
