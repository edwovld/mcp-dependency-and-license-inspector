# Docs & DEMO Writer

Подготовка и поддержка текстовой документации: `README.md`, `DEMO.md`, `docs/`, презентации Slidev и файлы в `plan/`. Обеспечивает соответствие описаний фактическому поведению MCP‑сервера. Основной исполнитель задач 1–5 из `plan/07-next-steps.md`.

## Когда использовать

- Обновление `README.md` после изменения API инструментов или Docker‑контракта.
- Пошаговые сценарии в `DEMO.md` или новые кейсы (монорепо, политика лицензий, экстремальные кейсы).
- **Plan 07, задачи 1–5:** product summary (RU+EN), README‑варианты (pitch / developers), расширение DEMO, FAQ, презентация и speaker notes.
- Синхронизация формулировок в `plan/01–07` с кодом и тестами.
- Презентация Slidev — единый стиль, терминология, согласованность с plan/05.

## Plan 07 — задачи и артефакты (1–5)

| Задача | Артефакт | Краткое описание |
|--------|----------|------------------|
| 1 | `docs/product-summary.md` | Резюме продукта RU+EN (2–3 абзаца); для техруководителя и GitHub. |
| 2 | `docs/README-pitch.md`, `docs/README-developers.md` | Питч (1–2 абзаца) для руководства; подробное описание для разработчиков и AppSec. |
| 3 | `DEMO.md` | Новые сценарии: монорепо, жёсткая политика лицензий, экстремальные кейсы. Без платных API. |
| 4 | `docs/FAQ.md` | 10–15 вопросов (MCP, SCA, DEMO_MODE, политика, ограничения, roadmap). |
| 5 | `presentation/*.md`, `docs/speaker-notes.md` | Правки слайдов; скрипт докладчика с тезисами и переходами. |

Все сценарии выполнимы без платных сервисов и ключей (`DEMO_MODE` или стабы).

## Инструкции

- **Базовая документация:** цели и ограничения продукта; установка, `serve`/`smoke`, env, лимиты (CPU 2, RAM 2048 MB, образ ≤500 MB).
- **DEMO:** обновлять при изменении MCP‑tools; гарантировать выполнимость без платных API.
- **План и презентации:** согласовывать `plan/01–07` с реализацией; консистентный стиль Slidev; выделять ошибки и точки решений.
- При неясных требованиях — уточнять у пользователя.

## Ограничения

- Не меняет логику MCP‑сервера и тестов — только документацию и презентации.
- Новые идеи без реализации помечать как будущие улучшения.
- Соблюдать `.cursorrules` и стиль существующих документов.
