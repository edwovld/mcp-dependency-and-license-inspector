# Ценностное предложение: MCP «Инспектор зависимостей и лицензий»

Чёткая формулировка: что делает сервер, для кого и какой эффект.

---

## One-pager (кратко)

**MCP «Инспектор зависимостей и лицензий»** — единая точка входа для анализа зависимостей и лицензий проекта прямо из ИИ-клиентов (Cursor, Claude Code, VS Code, Windsurf).
Сервер сканирует проект по нескольким экосистемам (JS/TS, Python, при необходимости JVM), проверяет уязвимости и лицензии, сверяет с политикой компании и выдаёт структурированные отчёты и SBOM.
Всё это доступно в диалоге с ИИ: один запрос — сводка рисков и конкретные рекомендации, без переключения на отдельные инструменты и веб-консоли.

**Для кого:** разработчики в AI-редакторах, инженеры по безопасности, юристы и compliance (через разработчиков или напрямую через ИИ).

**Эффект:** сокращение времени аудита с часов до минут, раннее выявление критичных CVE и недопустимых лицензий, готовность к регуляторным требованиям за счёт SBOM и отчётов,
удобные подсказки и команды прямо в чате.

---

## Что делает MCP

| Возможность | Описание |
|-------------|----------|
| **Единая точка входа** | Все запросы по зависимостям и лицензиям идут через один MCP-сервер из любого MCP-клиента. |
| **Кросс-языковой анализ** | Один проект — один граф зависимостей и один отчёт по JS/TS (npm, yarn, pnpm), Python (pip, poetry), при необходимости JVM (Maven, Gradle). |
| **Скан уязвимостей** | Интеграция с OSV, npm audit, pip-audit и др.; агрегированный список CVE с severity и рекомендуемыми версиями. |
| **Скан лицензий** | Сбор лицензий из метаданных пакетов и при необходимости из файлов; флаги риска (copyleft, неизвестная, множественная). |
| **Проверка политик** | Движок политик (YAML/JSON): допустимые/запрещённые лицензии, блокировка по CVE; отчёт о нарушениях и рекомендациях. |
| **SBOM и отчёты** | Генерация SBOM (SPDX, CycloneDX) и человекочитаемых отчётов (Markdown, JSON) для разработчиков, безопасников и юристов. |
| **Формат для ИИ** | Результаты в виде структурированных сводок, таблиц и приоритизированных списков — удобны для интерпретации в диалоге и генерации ответов. |

## Для кого

### Разработчики (в IDE / AI-редакторах)

- Проверить проект перед релизом или PR: «проверь зависимости и лицензии» — получают сводку и приоритизированный список действий.
- Понять, что обновить в первую очередь и какие риски несут текущие версии.
- Получить конкретные команды или предложения по патчам, а не только сырые данные.

### Инженеры по безопасности (AppSec / SecOps)

- Быстрый аудит нового или унаследованного репозитория без ручного запуска разных инструментов по языкам.
- Проверка соблюдения корпоративной политики лицензий (например, запрет GPL в ядре, разрешение в dev-инструментах).
- Запросы в стиле: «покажи все GPL-зависимости в продакшен-пути» или «сводка по критичным CVE».

### Юристы и Compliance

- Короткие агрегированные отчёты по лицензиям с объяснением рисков и вариантов (заменить, исключить, принять).
- ИИ как переводчик: из технических деталей (списки пакетов, SPDX ID) в понятные формулировки и рекомендации.

## Какой эффект

| Выгода | Описание |
|--------|----------|
| **Сокращение времени аудита** | Вместо часов ручного запуска npm audit, pip-audit, просмотра разных консолей — минуты диалога с MCP и ИИ: один запрос — полная картина. |
| **Снижение рисков** | Раннее обнаружение критичных CVE и недопустимых лицензий до мержа в основную ветку или до релиза; меньше юридических и репутационных последствий. |
| **Упрощение решений** | MCP выдаёт приоритизированные списки (must-fix / should-fix / nice-to-have) и рекомендации по заменам — проще планировать спринты и согласовывать с compliance. |
| **Улучшение DX** | Рекомендации приходят в контексте кода и чата, без переключения на отдельные веб-консоли и без необходимости помнить десяток CLI-команд. |
| **Готовность к регуляторам** | Автоматическая генерация SBOM и отчётов помогает соответствовать требованиям заказчиков и регуляторов (например, по прозрачности цепочки поставок). |

## Отличие от существующих решений

- **От других MCP:** не только «валлидатор одного пакета» (BoostSecurity) и не только одна экосистема (NPM Plus, Maven Tools, VulniCheck). Единый кросс-языковой аудит + лицензии + политики + SBOM
  в одном сервере.
- **От классических SCA (Snyk, Black Duck, FOSSA):** не замена, а дополнение: те же задачи (зависимости, уязвимости, лицензии) доступны в формате «вопрос–ответ» в ИИ-диалоге
  и при необходимости локально без обязательной привязки к SaaS.
- **От CLI/CI по отдельности:** один протокол (MCP), один набор инструментов для всех экосистем в проекте; результаты сразу пригодны для интерпретации ИИ и формулировки ответов пользователю.

## Краткий текст для README / презентации

**MCP «Инспектор зависимостей и лицензий»** даёт ИИ-ассистентам в Cursor, Claude Code, VS Code и других MCP-клиентах возможность за один запрос проанализировать зависимости проекта
по нескольким языкам (JavaScript/TypeScript, Python, JVM), проверить уязвимости и лицензии, сравнить с вашей политикой и выдать сводный отчёт или SBOM.
Разработчики получают конкретные рекомендации в чате;
безопасники и юристы — единый отчёт без ручного сбора данных из разных инструментов. Меньше времени на аудит, меньше рисков по CVE и лицензиям,
готовность к регуляторным требованиям.
